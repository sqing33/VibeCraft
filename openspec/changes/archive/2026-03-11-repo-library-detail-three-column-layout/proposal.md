## Why

Repo Library 仓库详情页当前仍按“记录详情”思路堆叠报告、快照、分析运行和卡片信息，首屏信息密度失衡，报告正文长期占据大块空间，但用户在完成解析后更常用的是快照上下文、技术栈摘要、卡片列表和卡片证据。

本变更将把仓库详情页重构为固定三栏知识工作台，并把正式报告第一部分里的“技术栈与模块语言”摘要结构化提取出来，直接服务于浏览和检索上下文，而不是要求用户反复阅读整篇报告。

## What Changes

- 将 Repo Library 仓库详情页重构为固定三栏布局：左栏展示快照/分析运行选择与技术栈摘要，中栏展示知识卡片列表，右栏展示卡片详情与可滚动 evidence。
- 将“快照”和“分析运行”从卡片列表改为下拉选择器，减少纵向占用，让单屏内同时容纳三栏主工作区。
- 从正式报告第一部分结构化提取技术栈摘要字段：生成时间、主要语言/技术栈总览、后端、前端、其它模块，并通过仓库详情接口返回。
- 把完整报告从主页面移除，改为“查看报告”按钮触发的弹窗/抽屉查看，保留全文访问但不占据常驻首屏。
- 调整三栏滚动行为：页面外层不滚动，中栏列表独立滚动，右栏 evidence 区独立滚动，左栏保持工具台式静态布局。

## Capabilities

### New Capabilities

- `repo-library-detail-workspace`: Repo Library 仓库详情页的三栏知识工作台布局、报告弹窗和分栏滚动交互。

### Modified Capabilities

- `repo-library-knowledge`: 正式报告中的技术栈与模块语言摘要需要被结构化提取并作为仓库详情上下文返回。

## Impact

- 后端：`backend/internal/repolib` 需要新增正式报告第一部分摘要提取逻辑，并在 Repo Library detail API 中返回结构化字段。
- 存储/API：Repo Library repository detail 响应模型需要新增技术栈摘要对象；同步流程需要在 report/cards 刷新时一并刷新该摘要。
- 前端：`RepoLibraryRepositoryDetailPage` 需要重构为固定三栏布局，并新增报告查看弹层与下拉选择交互。
