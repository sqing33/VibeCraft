## Context

当前详情页的问题不是信息过多，而是每一栏都被做成了相似重量的“独立卡片容器”。用户打开页面后，很难快速判断哪里是操作上下文、哪里是列表导航、哪里是主要阅读内容。

本次改动仅聚焦布局与样式层级，不改变现有数据流、接口和功能边界。

## Goals / Non-Goals

**Goals:**
- 降低左栏的视觉重量，把它从“控制台面板”收敛成“上下文摘要栏”。
- 让快照/分析运行切换更接近页头级工具，而不是详情正文的一部分。
- 让中栏明显承担导航职责，右栏明显承担阅读职责。
- 保持现有数据结构和滚动语义不变，只重分配视觉层级。

**Non-Goals:**
- 不更换现有的报告弹窗机制。
- 不增加新的筛选器、搜索框或卡片分组能力。
- 不改后端详情接口。

## Decisions

### 1. 将快照和分析运行提升到顶部上下文工具条

原因：
- 这两个控件本质是页面上下文切换，而不是左栏内容。
- 上移后左栏可以从“操作+摘要混合区”变成纯摘要区。

### 2. 左栏改为窄摘要栏

原因：
- 技术栈摘要是辅助理解，不该和主详情区争视觉中心。
- 左栏宽度缩小后，右栏的阅读空间会更自然。

方案：
- 左栏保留技术栈摘要和分析会话动作。
- 当前快照的 ref/commit 信息移到顶部上下文工具条。

### 3. 中栏改成轻量导航列表

原因：
- 卡片列表应该像目录，而不是一组与详情区等权的大内容块。

方案：
- 弱化边框和背景，强化选中态。
- 保留标题、类型、结论预览，但收缩装饰。

### 4. 右栏作为唯一主内容面板

原因：
- 用户最终关注的是卡片详情和证据，而不是三个并列的大框。

方案：
- 右栏保留重边距和内容卡感。
- Evidence 改成右栏底部支撑区，弱化其与正文的竞争关系。

## Risks / Trade-offs

- [Risk] 顶部工具条信息过多导致换行 → Mitigation：保留响应式换行，控件宽度固定且优先展示选择器。
- [Risk] 左栏过窄后技术栈文本可读性下降 → Mitigation：摘要字段允许多行，但弱化外层边框，减少视觉噪声。
- [Risk] 中栏过轻导致层级不足 → Mitigation：通过当前选中态和 hover 态保留清晰导航反馈。
