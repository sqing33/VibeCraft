## Context

设置弹窗目前已经按标签拆分功能，但各标签的 UI 密度和布局策略来自不同阶段的实现：有的页面把按钮放在内容顶部，有的页面使用较高的默认 HeroUI 控件，有的页面自己处理滚动。这让用户在不同设置标签之间切换时会感受到明显的不一致。

## Goals / Non-Goals

**Goals:**
- 给设置弹窗建立统一的视觉基线：胶囊 tab、紧凑输入、紧凑按钮、固定底栏。
- 尽量通过共享样式常量和共享布局组件减少重复实现。
- 保持现有设置功能与接口不变。

**Non-Goals:**
- 不重做设置项信息架构。
- 不修改非设置区域的全局组件样式。
- 不改动业务接口或字段定义。

## Decisions

### 1. 使用共享设置 UI 常量和布局容器
- 决策：新增设置页专用样式常量与布局组件，供各设置标签复用。
- 原因：这类统一风格跨越多个标签，局部复制 className 很难长期维持一致性。

### 2. 各标签采用“内容滚动 + 底部固定操作栏”
- 决策：标签根节点统一改成 `h-full/min-h-0/flex-col`，中间内容区滚动，底栏固定。
- 原因：用户在长表单中始终能看到主要操作，符合本轮统一诉求。

### 3. 仅在设置域内应用胶囊化样式
- 决策：不全局覆盖 HeroUI 主题，而是在设置域组件中显式传入紧凑样式。
- 原因：避免影响聊天、看板等其他页面的既有密度。

## Risks / Trade-offs

- [局部显式样式较多] → 通过共享常量/布局组件减少重复。
- [专家/技能等复杂标签可能仍有局部差异] → 优先统一顶层操作与主要表单控件，保留内容区内部必要差异。
