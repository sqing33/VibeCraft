## Context

详情页已经具备左侧上下文、右侧选卡与阅读的结构，但几个问题仍然明显：外层留白偏大，局部滚动被取消后页面整体过长，右侧选卡与详情仍像两个独立面板，页头时间信息也没有和仓库上下文形成清晰映射。

## Goals / Non-Goals

**Goals:**
- 减少四个主卡片在页面左侧与顶部的视觉留白。
- 让四个主卡片重新回到一个受控高度的工作区内。
- 让仓库上下文和卡片详情在超高时各自滚动，而不是整页无限展开。
- 将右侧选卡区与详情区合并为一个统一视觉容器。
- 移除重复说明文字和不必要标题，把生成时间提到页头。

**Non-Goals:**
- 不调整后端返回字段。
- 不修改搜索逻辑和卡片数据结构。
- 不改变报告弹窗行为。

## Decisions

### 1. 受控高度重新引入，但只让需要的内容区域滚动

原因：
- 用户需要四个主卡片稳定落在同一工作区中，而不是随着内容无限向下生长。
- 之前的问题不是“局部滚动错误”，而是“整列被拉伸后出现空白”。这次通过受控高度配合明确的内容滚动区来修正。

### 2. 右侧使用单一外层容器包住知识卡片选择带和详情区

原因：
- 视觉上它们属于同一条交互链路，分成两张独立卡片会显得断裂。
- 合并后，顶部切换和下方阅读会更像一个连续的知识查看器。

### 3. 页头展示生成时间，仓库上下文卡片去掉重复生成时间字段

原因：
- 生成时间是当前快照级全局状态，更适合作为页头上下文。
- 仓库上下文卡片内已经有多个字段分段，重复放生成时间只会增加噪声。

## Risks / Trade-offs

- [Risk] 受控高度过小会让局部滚动过于频繁 → Mitigation：仍按桌面视口高度计算工作区，并保留足够的详情可视面积。
- [Risk] 顶部与左侧留白收紧后页面可能显得拥挤 → Mitigation：只收紧外层留白，不压缩卡片内部核心内容间距。
- [Risk] 右侧合并后分区边界不清 → Mitigation：使用内部分隔线和不同背景层次区分选择带与详情区。
